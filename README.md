This repository provides the templates you will use 
for your final project reports.
Project reports that are not in compliance with these templates
will not be accepted.

It is recommended you divide writing your report into two parts:
1. Write the content
1. Format the content

It is more efficient to focus on deciding **what** will be in your report
before worrying about **how it looks**.
Doing so will potentially save you hours of formatting your final report.

Before you begin to format your paper,
first write and save the content
as a separate text file.
Complete all content
and organizational editing
before formatting.

The template files themselves also offer specific guidance 
about how to use them.
Be sure to read them carefully,
or you may end up "breaking" the templateâ€”especially
if you use Microsoft Word.

For example,
do not number the text headings
since the template will do that for you.

# Options for Writing the Report

This repository gives you two options for writing your report:
1. Microsoft Word (more familiar but error-prone)
1. LaTeX (recommended; difficult to learn, but robust and official)

## Option 1: Microsoft Word 

Use the `conference-template-a4.docx` file
to write your report. 
Note that this Word document comes with ready-made styles
for your use and convenience.

## Option 2: LaTeX 

LaTeX is the most professional option 
for your project reports
and is the standard format 
for many international conferences, 
journals, 
and other areas of research.
Writing your project report in LaTeX
will help you to be ready 
to present your work
in local and international conferences.

There are two ways to write LaTeX files:
using an online editor such as Overleaf,
or using a locally-installed editor.
We recommend you use Overleaf
for your project report.

### Using Overleaf (Recommended)

Create an Overleaf account
(if you haven't already)
and create a new project
using the template avalable at this link:
[https://www.overleaf.com/latex/templates/ieee-conference-template/grfzhhncsfqn](https://www.overleaf.com/latex/templates/ieee-conference-template/grfzhhncsfqn)

### Using a Local Editor & Compiler

Being able to locally write and compile LaTeX files into PDFs 
is a good skill to have
(especially in academic 
and research-oriented circles)
but can be time-consuming 
and technically complex.
Endeavor on this path at your own risk.

The files in the `latex` directory provides all the necessary files
for use in a local installation.

# Content Requirements

Your project report must be 40--50 pages long (or about 8,000--12,000 words) and contain the following sections:

| **    Element   **                    | **    Description   **                                                                                                                                                                                                                                                                 |
|---------------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
|     Abstract / Executive Summary      |     A   short summary of the entire project, including the problem, the solution, and   the key results.                                                                                                                                                                               |
|     Introduction                      |     Sets   the context, explains the problem being solved, and defines the project's   objectives and scope.                                                                                                                                                                           |
|     Literature Review / Background    |     Discusses   existing work or technologies related to the project.                                                                                                                                                                                                                  |
|     Methodology   / System Design     |     A   detailed description of how the project was done. This is   the core of the report. It explains the tools, technologies, algorithms, or   processes used. For example, in a software project, this would include   database designs, flowcharts, and architecture diagrams.    |
|     Implementation                    |     Describes   how the design was put into practice.                                                                                                                                                                                                                                  |
|     Testing   / Results               |     Presents   the outcomes of the project. This includes data, screenshots of the working   product, graphs, or analysis that proves the project works as intended.                                                                                                                   |
|     Conclusion and Future Work        |     Summarizes   what was achieved and suggests how the project could be extended or improved   in the future.                                                                                                                                                                         |
|     References   / Bibliography       |     A   list of all sources cited.                                                                                                                                                                                                                                                     |
|     Appendices                        |     Extra   material like code snippets, user manuals, or raw data.                                                                                                                                                                                                                    |
|     Recommendation   letters          |     Minimum   2: One from the Supervisor, the other one is from the Industry Representative                                                                                                                                                                                            |
